# 多类型回归测试用例（4 个单文件 HTML Demo）

用来检验「原型分类」和「六维评分卡区分度」是否合理。都是自包含单文件、无外部依赖，可直接在 Demo 详情页内嵌 iframe 里试玩。

## 怎么用

1. 打开站点（`http://localhost:3100`）→ 顶部 **提交 Demo** → 选 **上传单文件 HTML** → 选 `test-demos/` 里的某个 `.html` → 填标题/简介 → 提交。
2. 进该 Demo 详情页 → 点 **🔮 生成投资人估值**（约 1–2 分钟）。
3. 看 **archetype 分类** 是否正确、**六维雷达** 是否合理、点 **生成改进建议** 看建议是否贴合该类型。
4. 四个都做完后，跑 `node scripts/backtest-valuation.js` 看分原型聚合（注意：这些是合成 Demo、没有真实融资结果，回测样本仍需真实项目，见文末）。

## 预期分类与重点观察

| 文件 | 预期 archetype | 该类型重点信号（scoreRubric 会据此打分） | 重点验证 |
| --- | --- | --- | --- |
| `1-game-breakout.html` | **game** | 可玩性·留存·分享传播·ARPU | 建议应谈"可玩性/关卡/留存/传播"，moat 应偏低（打砖块是经典玩法、缺 IP/差异化） |
| `2-tool-json-formatter.html` | **tool** | 省时·使用频率·错误率·付费意愿 | 建议应谈"省时/使用频率/付费意愿"；这类是常见工具、护城河低 |
| `3-website-saas-landing.html` | **website**（或 saas，皆可接受） | 转化动作·线索·SEO·品牌文案 | 建议应谈"转化率/CTA/SEO/线索"，而不是游戏化 |
| `4-ecommerce-mini-shop.html` | **ecommerce** | GMV·订单·复购·支付闭环 | 建议应谈"支付闭环/复购/客单价/履约" |

## 判断"区分度合理"的标准

- **分类正确**：四个分别落到 game / tool / website / ecommerce（website↔saas 之间有歧义可接受，别串到游戏/工具即可）。
- **建议贴合类型**：游戏谈留存传播、工具谈省时频率、网站谈转化 SEO、电商谈 GMV 复购——而不是千篇一律。
- **诚实性**：因为都没有真实用户/收入，validation / commercial / moat 维度普遍偏低是**正确**的（评分卡缺数据从保守）。别期待高分；要看的是"相对侧重"是否随类型变化。

## 关于回测（重要）

这 4 个是**合成 Demo，没有真实融资/收购结果**，所以喂不进 `backtest-valuation.js` 的命中率计算（那需要项目有可核实的货币 outcome）。
要真正攒回测样本，需要**真实、可查证融资额的项目**：把它们以「在线 URL」形式提交、估值后，在详情页「报告真实进展」里填入**带来源链接**的融资额（verified）。
需要的话我可以另外给你列一批"公开融资额可查 + 有来源"的真实 AI 产品清单，供你录入做回测基线。
