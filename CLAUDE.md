# 项目说明(CLAUDE.md)

> 本文件会在每个新会话开始时自动加载。它让"换账号/换会话后无缝接手"变成自动行为。
> 可直接复制到任何其他项目根目录使用。

## 默认语言:简体中文

- 除非我另有要求,**始终用简体中文回复**(包括解释、说明、报告、提问等各种文本)。
- 代码、命令、变量名、文件名、路径、专有名词、API 名称、报错原文等**保持英文原文**,不要翻译。

## 会话开始时(自动接手)

1. 先检查本目录是否存在 `进度.md`。
2. 如果存在:**先完整读取 `进度.md`,以及它"关键文件"里列出的文件**,理解当前任务的
   全部上下文。
3. 然后用一两句话向我复述"我接下来准备做什么",等我确认后再继续。
4. 如果不存在 `进度.md`,正常开始即可。

## 工作期间(随时可接手)

- 每完成一个明显的阶段,或在我说"要换号/要结束"时,**主动更新 `进度.md`**。
- `进度.md` 至少包含:任务目标、已完成、进行中/下一步、关键文件及路径、重要决定与
  约定、待解决问题。写到"一个没看过本次对话的新会话也能照着接手"的程度。
- 模板见 `任务交接/进度模板.md`。

## 为什么这么做

Windows 版 Claude 的会话按账号隔离,换号后看不到上一个账号的对话。但**文件与账号
无关**:把进度写进 `进度.md`,任何账号的任何新会话读一下就能接着干。这样不依赖某次
对话,零风险、也不碰封号红线。

## 备注

- 保持 `进度.md` 简洁、最新即可,不需要长篇大论(也省 token)。
- 项目专属的其他约定(技术栈、命令、规范等)可以往本文件后面补充。
