# 持续校准 SOP · 把估值校准变成活的"可信度看板"

> 目的：让"AI 估值 vs 真实结果"的对照**随时间持续积累**，沉淀成本站独家、不可复制的信任资产。
> 这是护城河的核心：算法可被抄，但"我们当初怎么估、后来真实怎样、敢公开承认误差"的长期记录抄不走。

## 一条铁律：估值必须"盲"

校准只有在**估值发生在结果公开之前**时才成立。所以：

- 用户提交项目时即时估值（此时还没有结果）→ 天然是盲的，这是正常路径。✅
- **记录真实结果(outcome)之后，不要再对该项目"强制重算"**。因为融资/收购类结果会被算进 `floor`、把答案泄漏进新估值，那条就不再是预测，而是抄答案。
- 回测/看板读的是**历史 `val_summary` / `valuation_history`**（带时间戳与 `algo_version`），即"当初那次盲估"。保持原样别覆盖。

## 数据从哪来

1. **创始人自报**（主力）：详情页「报告真实进展」(OutcomeReporter) 录入融资/营收/被收购/用户数/停运。**必须带来源链接**；只有可核实(`verified=1` 或带 `source_url`)的货币结果才进 `floor` 和看板。
2. **你/运营定期巡检**：用候选脚本找出"估值已久、还没结果"的项目，主动去查证后补录（带来源）。
3. **半自动辅助（可选）**：可参照 `enrich-comparables.js` 的做法（Tavily 搜索 + `extractFunding` 严格防编造）做"候选项目是否已融资"的初筛，但**最终必须人工确认来源**再入库。

## 月度巡检流程（建议每月一次，~15 分钟）

1. 跑 `node scripts/calibration-candidates.js 30`
   → 列出"估值已超 30 天、仍无真实结果"的项目（按估值额从高到低）。
2. 挑其中**估值高 / 你印象里有动静**的几个，去查有没有真实进展：
   - 融资/被收购 → 找到官宣或权威报道链接。
   - 营收(ARR/MRR) → 公开的 openstartup 页 / 创始人公开数据。
   - 停运 → 站点关停/公告。
3. 有结果的，在该项目详情页「报告真实进展」**带来源链接**录入（货币统一折人民币）。
4. 跑 `node scripts/audit-outcomes.js` 抽查新录入有没有明显错值。
5. 打开 `/calibration` 看板：命中率、中位比、按算法版本/原型对比，会随每条新结果更新。

## 怎么读看板（保持诚实）

- **价值类**(收购价/融资估值≈价值)：看真实是否落进预测区间、中位比是否趋近 1。
- **营收类**(ARR≠估值)：看隐含倍数(预测÷ARR)是否落在合理带[2–20×]。
- **算法版本对比**：每次调 prompt/权重打新 `algo_version`，看新版在"后续真实结果"上的命中是否更好——**只保留让命中变好的改动**。
- **公开误差**：看板要敢展示没命中的项目。承认误差本身就是可信度来源；藏起来反而摧毁它。

## 别做的事（会摧毁可信度）

- ❌ 记录结果后重估、再把"高命中"当政绩——这是抄答案。
- ❌ 录入无来源的自报大额结果去抬 floor/看板。
- ❌ 用知名标的"联网读数"刷好看的命中率（那测的是会上网，不是会预测）。
- ❌ 为了让某类（如游戏）历史离群点好看而上调该类估值——会把大量会失败的同类项目集体高估。

## 相关文件

| 文件 | 作用 |
| --- | --- |
| `scripts/calibration-candidates.js` | 列出"已估值待回填结果"的项目（月度巡检入口，只读） |
| `scripts/audit-outcomes.js` | 审计/清理可疑 outcome（只读 + 安全删单条） |
| `scripts/backtest-valuation.js` | 回测：价值类比区间、营收类看隐含倍数；`rerun` 用当前算法重估 |
| `app/calibration/page.js` | 公开可信度看板（命中率/中位比/版本·原型对比） |
| `components/OutcomeReporter.js` | 详情页录入真实结果（带来源、币种换算） |
| `lib/queries.js` `calibrationData()` | 看板数据 + 分组 |

## 未来可加（规模上来后）

- 自动巡检与提醒（项目估值满 N 天且无结果 → 提示运营/创始人补录）。
- 公开"可信度看板"页对外展示（预测 vs 现实的长期准确率），作为对投资人/用户的信任证明。
- 结果类型扩展：ProductHunt 排名、应用商店在架/评分、营收里程碑等更多可核实维度。
