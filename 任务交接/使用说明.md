# 换账号也能无缝继续任务 —— 交接方法

## 为什么用这个方法

Windows 版 Claude 现在是 MSIX 版,会话是**按账号隔离**的:换账号后,前一个账号
的那次对话在界面里看不到了(文件其实还在硬盘上,只是不显示)。所以"让 B 账号接着
打开 A 账号那次对话"做不到。

但有个关键事实:**你的工作成果(文件)跟账号无关**。比如这个文件夹里的东西,哪个
账号登录都在。所以只要把"任务进度"也写进一个文件,换号后开个新会话读一下,就能
接着干——这就是下面这套方法。

零风险、不动 Claude 任何数据、不碰封号红线。

---

## 三步用法

### 第 1 步(可选,养成习惯):让 Claude 边做边记进度

在任务进行中,时不时对 Claude 说一句(或在每完成一块时):

> 把当前任务进度更新到 `进度.md`。

让进度文件始终保持最新。`进度.md` 的结构见同目录的 `进度模板.md`。

### 第 2 步:**切换账号前**,先存一次进度

在你准备退出当前账号之前,把下面这段**整段复制**发给当前的 Claude:

```
我马上要换一个账号继续同一个任务。请把当前任务的完整进度写入/更新 进度.md,包含:
1. 任务目标
2. 已经完成的部分
3. 当前正在做什么(做到哪一步)
4. 下一步要做什么
5. 涉及的关键文件及它们的路径
6. 重要的决定、约定、限制条件
7. 还没解决的问题或坑
写得让一个完全没看过本次对话的新会话,也能照着 进度.md 无缝接手。
```

### 第 3 步:**换号登录后**,在新会话里接手

换好账号、连上**同一个文件夹**后,新开一个会话,把下面这段**整段复制**发过去:

```
我换了账号,继续同一个任务。请先读取 进度.md,以及它里面提到的相关文件,完整理解
上下文。然后用一两句话告诉我你接下来准备做什么,等我确认后再继续推进。
```

Claude 读完进度文件就能接上,确认一下就继续干。

---

## 小贴士

- **进度文件放在你正在做这个任务的文件夹里**(跟着任务走),不要只放一个地方。
- 每个任务一个 `进度.md`;任务复杂时,进度里多写"为什么这么决定",换号后更不容易跑偏。
- 真正的产出(代码、表格、文档等)本来就以文件形式留在文件夹里,换号不会丢。
- 如果你担心忘了存进度:养成"每做完一块就让 Claude 更新 进度.md"的习惯,这样任何
  时候被中断、被限额,都能随时接手。

---

## 关于"绕额度"的提醒

如果换号的目的是**靠多个号绕过用量限制**,这本身可能触碰 Anthropic 的使用政策、有
封号风险(跟用什么方法无关)。上面这套交接法只是帮你在**合法、不同用途的账号之间**
延续工作,请按自己的实际情况使用。
